I have just "discovered" OpenCL and i am totally excited about the idea of using it in combination with the simulation nodes :D Atm the these node trees are using "classic" multithreading on the CPU and an ad-hoc batch processing model: data from the scene (currently only vertex positions and some basic particle attributes) is copied into fixed-size batches, which are then processed per node by available worker threads. It is already very efficient as far as one can tell from the simple arithmetic nodes i made so far, but i'd gladly replace this with OpenCL for increased efficiency and better parallel design. Currently i'm further improving the node interface capabilities to combine them with the RNA, so that basically all the properties in Blenders API can be used inside the node trees with a generic GetData/SetData node.
